02
Le concept

Une fonction, c'est une machine à café.

Métaphore filée — La machine à café

Vous mettez des ingrédients (grains, eau, sucre) → la machine fait son travail interne → vous récupérez un produit (un café). Vous n'avez pas besoin de comprendre la mécanique : vous appuyez sur le bouton, ça sort.

Une fonction PHP, c'est pareil. Vous lui donnez des paramètres (les ingrédients), elle calcule, et elle vous retourne un résultat. Vous pouvez l'utiliser autant de fois que vous voulez sans la refabriquer.

Définition technique

Une fonction est un bloc de code nommé, écrit une seule fois, qu'on peut appeler autant de fois que nécessaire avec différentes valeurs d'entrée. Elle prend zéro, un ou plusieurs paramètres, et peut retourner une valeur via le mot-clé return.

Trois bénéfices concrets :

  • Réutilisabilité — écrit une fois, utilisé partout.
  • Maintenabilité — un seul endroit à modifier pour tout changer.
  • Lisibilité — le code raconte ce qu'il fait : calculerTVA($prix) est plus parlant que 15 lignes de calcul.
▶ Streaming · Spotify

Une fonction utilisée des millions de fois par jour

Chez Spotify, quand un utilisateur se connecte, le système appelle verifierIdentifiants($email, $password). Cette même fonction est utilisée sur la page web, l'app iOS, l'app Android, et l'API publique. Si Spotify améliore la sécurité, une seule fonction à modifier — pas 4 codes-bases à mettre à jour.

03
L'anatomie

Le mot-clé function, ligne par ligne.

En PHP, on déclare une fonction avec le mot-clé function, suivi d'un nom, de parenthèses (qui contiennent les paramètres), puis d'accolades (qui contiennent le code). Voici la version la plus simple, décortiquée.

fonction_simple.php
01 <?php
On ouvre le bloc PHP. Rappel : tout ce qui est entre <?php et ?> s'exécute sur le serveur.
02 function direBonjour() {
Le mot-clé function déclare la fonction. direBonjour est son nom. Les () vides signifient : zéro paramètre. Le { ouvre le corps de la fonction.
03 echo "Bonjour tout le monde !";
Le corps de la fonction. C'est ce qui s'exécute à chaque appel. Ici, on affiche une chaîne. Note : à ce stade, la fonction est définie mais pas encore appelée.
04 }
Le } ferme le corps. La fonction est maintenant en mémoire, prête à être utilisée.
05
06 direBonjour(); // Appel n°1
L'appel. On écrit le nom de la fonction suivi de (). PHP saute dans la fonction, exécute son corps, et revient ici. Affiche Bonjour tout le monde !
07 direBonjour(); // Appel n°2
Deuxième appel. On peut en faire 100, 1000. C'est tout l'intérêt : une fonction est écrite une fois, appelée à volonté.
08 ?>
On ferme le bloc PHP.
Le mot-clé est bien function, pas autre chose

D'autres langages utilisent def (Python), func (Go), ou method (Ruby). En PHP, c'est function et seulement function. Toute autre forme déclenche une erreur de syntaxe.

→ Question 12 du QCM porte exactement là-dessus.

Paramètres et valeur de retour

Une fonction sans paramètre est rare. Le plus souvent, on lui donne des données à traiter, et on récupère un résultat avec return.

fonction_retour.php
01 function calculerSomme($a, $b) {
La fonction calculerSomme attend deux paramètres : $a et $b. Ces noms sont des variables locales à la fonction.
02 return $a + $b;
return renvoie une valeur à l'endroit où la fonction a été appelée. Attention : return n'affiche rien à l'écran, il donne juste un résultat.
03 }
Fin de la fonction.
04
05 $total = calculerSomme(5, 3);
On appelle la fonction avec les arguments 5 et 3. La valeur retournée (8) est rangée dans $total. Distinction : $a, $b sont les paramètres, 5, 3 sont les arguments.
06 echo "Total : $total";
Affiche Total : 8. La fonction a calculé, le code appelant affiche. Séparer les deux est une bonne pratique.
La nuance fondamentale : echo vs return

echo affiche dans la page (visible par l'utilisateur). return renvoie une valeur au code qui a appelé la fonction (invisible tant qu'on n'en fait rien).

Une fonction qui ne fait que echo est moins flexible : on ne peut pas réutiliser son résultat. Une fonction qui return peut être enchaînée, stockée, comparée. Réflexe : préférer return et laisser le code appelant décider quoi faire du résultat.

Paramètres par défaut

Parfois, un paramètre a une valeur « habituelle » qu'on souhaite ne pas re-passer à chaque appel. On peut lui donner une valeur par défaut dans la déclaration de la fonction. Si l'appel ne fournit pas l'argument, PHP utilise la valeur par défaut.

parametres_defaut.php
01 function saluer($nom = "Visiteur") {
$nom = "Visiteur" donne une valeur par défaut. Si l'appel ne précise rien, $nom vaudra automatiquement "Visiteur".
02 echo "Bonjour $nom !";
Corps classique. Affichera le nom passé, ou la valeur par défaut.
03 }
04
05 saluer("Alice"); // Bonjour Alice !
Argument fourni → la valeur par défaut est écrasée par "Alice".
06 saluer(); // Bonjour Visiteur !
Aucun argument → PHP utilise la valeur par défaut. Pas d'erreur, parce que le paramètre est facultatif.
Les paramètres par défaut doivent être à droite

Tous les paramètres sans valeur par défaut doivent être déclarés avant ceux qui en ont une. Sinon PHP ne saurait pas auquel correspond un argument manquant.

Mauvais : function f($x = 0, $y) { ... } — erreur de logique.
Bon : function f($y, $x = 0) { ... }

04
Ranger les données

Trois façons de regrouper plusieurs valeurs.

Une variable contient une seule donnée. Un tableau (array en anglais) en contient plusieurs, regroupées sous un seul nom. PHP propose trois variantes selon l'organisation voulue : indexé (liste ordonnée), associatif (paires clé/valeur), multidimensionnel (tableau de tableaux).

4.1 — Tableau indexé : le wagon

Métaphore — Le wagon de train

Imaginez un train avec des compartiments numérotés : 0, 1, 2, 3... Vous demandez « donne-moi le contenu du compartiment 2 » et on vous le sort. Un tableau indexé fonctionne exactement comme ça.

Attention : en informatique, on commence à compter à 0, pas à 1. Le premier compartiment est [0].

tableau_indexe.php
01 $fruits = ["Pomme", "Banane", "Orange", "Fraise"];
Création d'un tableau avec les crochets [ ]. Les valeurs sont séparées par des virgules. PHP attribue automatiquement les index 0, 1, 2, 3.
02 echo $fruits[0]; // Pomme
On accède à un élément avec $tableau[index]. L'index 0 donne le premier élément, pas le second.
03 echo $fruits[1]; // Banane
Index 1 → deuxième élément. C'est le piège classique du QCM.
04 $fruits[1] = "Mangue";
Modifier un élément existant. Le wagon n°1 contient maintenant "Mangue" à la place de "Banane".
05 $fruits[] = "Kiwi";
Ajout à la fin. Les crochets [] vides signifient « prends le prochain index libre ». Ici, "Kiwi" devient $fruits[4].
06 echo count($fruits); // 5
count() retourne le nombre d'éléments du tableau. C'est la fonction standard en PHP (pas size, ni length, ni total).
07 foreach ($fruits as $fruit) {
Pour parcourir tous les éléments, on utilise foreach. Rappel Jour 1 : $fruit prend successivement chaque valeur du tableau.
08 echo "- $fruit<br>";
À chaque tour, on affiche le fruit courant suivi d'un saut de ligne HTML.
09 }
Fin de la boucle.

4.2 — Tableau associatif : le répertoire

Métaphore — Le répertoire téléphonique

Dans votre répertoire, vous ne cherchez pas « le contact n°47 », vous cherchez « Marie ». Chaque nom (la clé) donne un numéro (la valeur). Les clés ne sont pas des chiffres : ce sont des mots que vous avez choisis.

C'est exactement ça, un tableau associatif. La clé est descriptive : "nom", "age", "email"...

$personne
"nom" "Dupont" →
"prenom" "Marie" →
"age" 28 →
"ville" "Paris" →
tableau_associatif.php
01 $personne = [
Même syntaxe que pour l'indexé : crochets [ ]. La différence est à l'intérieur.
02 "nom" => "Dupont",
Une entrée : clé => valeur. La flèche => est le symbole d'association. On lit : « la clé "nom" vaut "Dupont" ».
03 "prenom" => "Marie",
Chaque paire est séparée par une virgule.
04 "age" => 28,
Les valeurs peuvent être de n'importe quel type (string, int, float, bool, autre tableau...).
05 "ville" => "Paris"
La virgule finale est facultative en PHP (mais autorisée).
06 ];
Fin du tableau.
07
08 echo $personne["nom"]; // Dupont
Accès par clé, entre guillemets. $personne["nom"] donne "Dupont".
09 $personne["email"] = "marie@x.fr";
Ajout d'une nouvelle clé. Si la clé n'existe pas encore, PHP la crée. Si elle existait, sa valeur serait écrasée.
10 foreach ($personne as $cle => $val) {
Variante de foreach avec clé ET valeur. À chaque tour, $cle et $val prennent une paire.
11 echo "$cle : $val<br>";
Affiche par exemple nom : Dupont, puis prenom : Marie, etc.
12 }
Indexé vs associatif : la différence se voit aux clés

Indexé → clés numériques automatiques (0, 1, 2...). On range une liste où l'ordre compte.

Associatif → clés nommées (chaînes choisies par vous). On range un objet où chaque info a un sens.

05
La boîte à outils

PHP offre des fonctions toutes faites pour les tableaux.

Pas besoin de réinventer la roue : PHP fournit une grosse bibliothèque de fonctions natives pour manipuler les tableaux. Voici les sept à connaître pour le QCM et la vraie vie.

fonctions_tableaux.php
01 $nombres = [5, 2, 8, 1, 9];
Notre tableau de départ.
02
03 count($nombres); // 5
count() → nombre d'éléments. Réflexe QCM : ni size, ni length, ni total.
04 sort($nombres); // [1, 2, 5, 8, 9]
sort() trie en place, par ordre croissant. Le tableau original est modifié. Pas de valeur de retour utile.
05 rsort($nombres); // [9, 8, 5, 2, 1]
rsort() trie par ordre décroissant (r pour reverse).
06 in_array(5, $nombres); // true
in_array(valeur, tableau) → retourne true si la valeur est dans le tableau, false sinon. Utile pour les tests de présence.
07 array_push($nombres, 10); // ajoute 10 à la fin
array_push() ajoute un (ou plusieurs) élément(s) à la fin du tableau. Équivalent de $nombres[] = 10;.
08 array_pop($nombres); // retire le dernier
array_pop() retire et renvoie le dernier élément. Push et pop fonctionnent comme une pile : on empile en haut, on dépile par le haut.
09 $a = [1,2]; $b = [3,4];
Deux tableaux à fusionner.
10 array_merge($a, $b); // [1, 2, 3, 4]
array_merge() fusionne plusieurs tableaux en un seul. Retourne le résultat — il n'altère pas les originaux.
11 array_slice($a, 1, 1); // [2]
array_slice(tab, début, longueur) extrait une portion du tableau. Ici : à partir de l'index 1, prendre 1 élément.
Le piège array_push + array_pop

Si vous faites array_push($t, 40) puis array_pop($t) sur $t = [10, 20, 30] :

  • Après push : [10, 20, 30, 40]
  • Après pop : [10, 20, 30] — on retire le dernier (40), pas le premier.

Le tableau revient à son état initial. → Question 45 du QCM.

06
Le texte

Manipuler du texte : compter, transformer, découper.

Une chaîne (string) n'est pas un bloc figé : on peut la mesurer, changer sa casse, en remplacer une partie, la couper en morceaux. Voilà les fonctions essentielles.

6.1 — Concaténer et compter

chaines_basics.php
01 $mot = "Bonjour";
Une chaîne simple. 7 caractères : B-o-n-j-o-u-r.
02 $age = 25;
Un entier, pour l'exemple de concaténation suivant.
03 echo "Age : " . $age; // Age : 25
Concaténation avec le . (le point). C'est l'opérateur PHP pour coller des chaînes. Pas le + (qui ferait une addition).
04 echo strlen($mot); // 7
strlen() compte le nombre de caractères de la chaîne. Espaces inclus. Pour "Bon jour", ce serait 8.
L'opérateur de concaténation est ., pas +

Habitude JavaScript : "Bonjour " + name. En PHP, c'est "Bonjour " . $name avec un point. Le + appliqué à des chaînes ferait une addition et donnerait souvent un résultat absurde ou une erreur.

→ Question 17 du QCM porte exactement là-dessus.

6.2 — Changer la casse, remplacer, nettoyer

chaines_transform.php
01 $texte = "Bonjour tout le monde";
02 strtoupper($texte); // BONJOUR TOUT LE MONDE
strtoupper() → tout en MAJUSCULES (uppercase).
03 strtolower($texte); // bonjour tout le monde
strtolower() → tout en minuscules.
04 ucfirst("bonjour"); // Bonjour
ucfirst() → met la première lettre en majuscule (upper case first).
05 ucwords($texte); // Bonjour Tout Le Monde
ucwords() → met chaque mot avec une majuscule initiale.
06 str_replace("monde", "univers", $texte);
str_replace(cherche, remplace, dans) → remplace toutes les occurrences. Donne "Bonjour tout le univers".
07 trim(" coucou "); // "coucou"
trim() → supprime les espaces (et tabs, retours à la ligne) au début et à la fin. Indispensable pour nettoyer une saisie utilisateur.

6.3 — Découper et fusionner

explode_implode.php
01 $phrase = "Pomme,Banane,Orange";
Une chaîne avec des séparateurs.
02 $fruits = explode(",", $phrase);
explode(séparateur, chaîne) → découpe la chaîne et retourne un tableau. Ici : ["Pomme", "Banane", "Orange"].
03
04 $liste = ["Alice", "Bob", "Charlie"];
Un tableau à recoller.
05 implode(" - ", $liste); // "Alice - Bob - Charlie"
implode(colle, tableau) → fait l'inverse d'explode : recolle les éléments avec un séparateur.
06
07 $email = "contact@ecoleit.fr";
08 substr($email, 8); // "ecoleit.fr"
substr(chaîne, début [, longueur]) → extrait une portion à partir d'un index. Ici, à partir du 8ᵉ caractère (le e de ecoleit) jusqu'à la fin.
09 substr($email, 0, 7); // "contact"
Avec 3 arguments : début et longueur. Ici : à partir de 0, 7 caractères.

6.4 — Le piège des guillemets : ' vs "

Guillemets simples ≠ guillemets doubles

Guillemets simples ' ' → contenu pris littéralement. Les $variables ne sont pas remplacées.

Guillemets doubles " " → PHP interprète les variables $var et les séquences d'échappement (\n, \t).

→ Question 18 du QCM porte exactement là-dessus.

Résultat
Pourquoi
$nom = "Marie";

Quand utiliser quoi ?

Doubles dès qu'il y a une variable à insérer : "Bonjour $nom". Plus court et plus lisible que la concaténation.

Simples pour du texte pur sans variable : 'Une chaîne fixe'. Microscopiquement plus rapide (PHP ne lance pas le parseur d'interpolation), mais surtout plus honnête : le lecteur sait qu'il n'y a rien à substituer.

07
La persistance

Lire et écrire dans un fichier, comme un cahier.

Une variable disparaît à la fin du script. Pour qu'une donnée survive entre deux requêtes, il faut l'écrire quelque part : dans un fichier, ou plus tard dans une base de données (Jour 3). Aujourd'hui : les fichiers.

Métaphore — Le cahier d'écolier

Un fichier, c'est un cahier. Vous pouvez : l'ouvrir, le lire, écrire dedans (en effaçant tout ou en ajoutant à la suite), et le refermer. PHP propose deux niveaux d'outils : les fonctions simples (file_get_contents, file_put_contents) qui font tout d'un coup, et les fonctions bas-niveau (fopen, fwrite, fclose) qui demandent d'ouvrir/refermer explicitement.

7.1 — Lire un fichier

lecture.php
01 if (file_exists("notes.txt")) {
file_exists() teste si le fichier existe. Réflexe : toujours vérifier avant de lire, sinon erreur.
02 $contenu = file_get_contents("notes.txt");
file_get_contents() → lit tout le fichier d'un coup et retourne son contenu sous forme de chaîne. La méthode la plus simple pour lire.
03 echo $contenu;
On affiche tel quel.
04 }
05
06 // Variante : lire ligne par ligne
07 $lignes = file("notes.txt");
file() → retourne un tableau dont chaque entrée est une ligne du fichier. Pratique pour traiter ligne par ligne.
08 foreach ($lignes as $n => $l) {
On parcourt avec l'index pour afficher le numéro.
09 echo "L.$n : $l<br>";
Numéro de ligne + contenu.
10 }

7.2 — Écrire dans un fichier

ecriture.php
01 $texte = "Mon nouveau contenu";
02 file_put_contents("sortie.txt", $texte);
file_put_contents(fichier, contenu) → écrit le contenu dans le fichier. Si le fichier existe, son contenu est écrasé. Si non, le fichier est créé.
03
04 // Ajouter à la suite, sans écraser
05 file_put_contents("sortie.txt", "\nLigne 2", FILE_APPEND);
Le troisième paramètre FILE_APPEND change tout : au lieu d'écraser, on ajoute à la fin du fichier existant. Indispensable pour les logs.
Sans FILE_APPEND, vous écrasez tout

Le piège classique : file_put_contents("logs.txt", "nouveau message"); remplace l'intégralité du fichier par cette seule ligne. Pour ajouter sans détruire, le drapeau FILE_APPEND est obligatoire.

→ Question 21 du QCM porte exactement là-dessus.

7.3 — La méthode bas-niveau : fopen

Pour des fichiers volumineux ou des opérations fines (lecture par chunks, position de curseur), on préfère ouvrir explicitement avec fopen. Le deuxième paramètre — le mode — décide ce qu'on peut faire.

fopen.php
01 $f = fopen("donnees.txt", "w");
fopen(chemin, mode) ouvre un fichier et retourne un handle (poignée) qu'on stocke dans $f. Mode "w" = écriture, fichier vidé.
02 fwrite($f, "Nouvelle ligne");
fwrite(handle, contenu) écrit dans le fichier ouvert.
03 fclose($f);
fclose(handle) ferme le fichier. Toujours fermer ce qu'on a ouvert (sinon le fichier reste verrouillé).
Mode Effet Fichier inexistant ?
'r' Lecture seule, curseur au début. Erreur.
'w' Écriture seule, fichier vidé. Créé.
'a' Écriture, curseur à la fin (append). Créé.
'r+' Lecture + écriture, curseur au début. Erreur.
Le mode 'w' détruit ce qui était là

Si vous ouvrez en mode 'w' un fichier qui contenait 1000 lignes importantes, tout est effacé instantanément, avant même d'écrire quoi que ce soit. Pour ajouter à la suite, c'est le mode 'a' (append).

→ Question 48 du QCM porte exactement là-dessus.

Mise en situation

WebBlog : un système de logs en 6 lignes.

On combine maintenant tout ce qu'on a vu : une fonction réutilisable, qui prend un message en paramètre, ajoute la date et écrit dans un fichier de logs. C'est exactement ce qu'utilisent Amazon, Netflix ou Cdiscount pour tracer les actions de leurs millions d'utilisateurs.

logs.txt
[2025-12-06 14:15] Utilisateur connecté
[2025-12-06 14:18] Page produit consultée
[2025-12-06 14:22] ERREUR — paiement échoué
logs.php
01 function ajouterLog($message) {
Une fonction qui prend un seul paramètre : le message à logger.
02 $date = date("Y-m-d H:i:s");
date() est une fonction native PHP. "Y-m-d H:i:s" = format année-mois-jour heure:minute:seconde. Donne par exemple 2025-12-06 14:15:00.
03 $ligne = "[$date] $message\n";
On construit la ligne complète avec interpolation (guillemets doubles pour que $date et $message soient remplacés). Le \n ajoute un saut de ligne.
04 file_put_contents("logs.txt", $ligne, FILE_APPEND);
Ajout à la suite avec FILE_APPEND. Si on oubliait ce drapeau, chaque appel écraserait le précédent.
05 }
06
07 ajouterLog("Utilisateur connecté");
Première utilisation. Une ligne est ajoutée à logs.txt.
08 ajouterLog("Page produit consultée");
Deuxième appel, deuxième ligne. La date est recalculée à chaque fois.
09 ajouterLog("ERREUR - paiement échoué");
On peut logger tout type de message. En vrai projet, on rangerait par niveau (INFO, WARN, ERROR).
Ce qu'on vient de combiner

Une fonction réutilisable (Ch. 03) + une chaîne interpolée (Ch. 06) + une écriture fichier en append (Ch. 07) + une fonction native date(). Six lignes de code qui résolvent un problème de production réel.

▼ E-commerce · Amazon

Pourquoi tous les gros sites font ça

Sur Amazon, chaque connexion, chaque clic, chaque erreur produit une ligne de log. Les développeurs analysent ensuite ces logs pour repérer les bugs et optimiser les temps de réponse. Si en pleine saison Black Friday une fonction de paiement ralentit, c'est dans les logs qu'on le voit en premier. Pas de logs, pas de visibilité. Pas de visibilité, pas de qualité.

Anti-pièges

Réflexes pour le QCM : 16 cartes à connaître.

Chaque carte ci-dessous condense un piège qui revient dans le QCM. Lisez-les avant l'épreuve : si vous bloquez sur une, retournez à la section correspondante.

QCM Q12 · #1
Le mot-clé est function

Pas def, pas func, pas method. Toujours function nomFonction().

QCM Q13/Q43 · #2
return ≠ echo

return renvoie au code appelant. echo affiche. Une fonction qui doit produire un résultat utilise return.

QCM Q44 · #3
Paramètre par défaut

function f($x = "défaut"). Si l'appel ne fournit pas l'argument, PHP utilise la valeur par défaut. Pas d'erreur.

QCM Q14 · #4
Compter = count()

Pas size(), ni length(), ni total(). Uniquement count($tab).

QCM Q15 · #5
Indexé vs associatif

Indexé = clés numériques auto. Associatif = clés nommées que vous choisissez ("nom" => "Marie").

QCM Q16/Q45 · #6
array_push ajoute à la fin

Et array_pop retire à la fin. Push + pop sur un tableau = retour à l'état initial.

QCM Q46 · #7
Accès associatif

$pays["capitale"] donne la valeur associée à la clé "capitale". Guillemets autour de la clé.

QCM Q17 · #8
Concaténer avec .

Pas + (JavaScript), pas &, pas ||. En PHP, c'est le point.

QCM Q18 · #9
Guillemets doubles = interpolation

"Bonjour $nom" remplace. 'Bonjour $nom' affiche le $nom textuellement.

QCM Q19 · #10
explode → tableau

explode(" ", "PHP is great") donne ["PHP", "is", "great"]. Pas une chaîne avec tirets.

QCM Q47 · #11
strlen compte tout

Espaces inclus. strlen("Bonjour") = 7. Pas 5, pas 8.

QCM Q20 · #12
Lire un fichier entier

file_get_contents() est la méthode la plus simple. Une seule ligne, retourne tout le contenu.

QCM Q21 · #13
FILE_APPEND ajoute

Sans ce drapeau, file_put_contents écrase. Avec ce drapeau, il ajoute à la fin.

QCM Q48 · #14
Mode 'w' = écrasement

fopen(..., 'w') vide instantanément le fichier. Pour ajouter, c'est 'a'.

Bonus · #15
Index commence à 0

Premier élément = [0]. Donc $fruits[1] = deuxième élément (rappel Jour 1).

Bonus · #16
Toujours fermer ce qu'on ouvre

Après fopen(), on appelle toujours fclose(). Sinon le fichier reste verrouillé.

Mise à l'épreuve

À vous de jouer. Six questions, vraies conditions.

Six questions tirées directement du QCM officiel. Cliquez sur la réponse que vous pensez correcte — verdict immédiat et explication.

Quel mot-clé PHP permet de définir une fonction personnalisée ?
Réponse D. En PHP, on déclare une fonction avec function nomDeLaFonction(). def appartient à Python, func à Go, method n'existe nulle part comme mot-clé de déclaration standalone.
Quelle est la différence fondamentale entre un tableau indexé et un tableau associatif en PHP ?
Réponse B. L'indexé est ["Pomme", "Banane"] (PHP attribue les index 0, 1 automatiquement). L'associatif est ["nom" => "Marie"] (clés nommées que vous choisissez). Les deux peuvent stocker n'importe quel type de valeur.
Quel opérateur PHP permet de concaténer deux chaînes de caractères ?
Réponse A. En PHP, on colle des chaînes avec le point : "Bonjour " . $nom. Le + appliqué à des chaînes ne fonctionne pas comme en JavaScript — il ferait une addition numérique.
Quelle est la différence essentielle entre guillemets simples et doubles pour les chaînes en PHP ?
Réponse B. "Bonjour $nom" affiche « Bonjour Marie ». 'Bonjour $nom' affiche littéralement « Bonjour $nom ». Pareil pour \n (saut de ligne réel dans les doubles, deux caractères dans les simples).
Quel est le rôle du drapeau FILE_APPEND utilisé comme troisième paramètre de file_put_contents ?
Réponse A. Sans FILE_APPEND, chaque appel écrase tout le fichier. Avec FILE_APPEND, le nouveau contenu est ajouté à la fin du contenu existant. Indispensable pour les logs.
Quel sera le contenu du tableau après ce code ?
$nums = [10, 20, 30];
array_push($nums, 40);
array_pop($nums);
Réponse A. Après push : [10, 20, 30, 40]. Après pop (qui retire le dernier, soit 40) : [10, 20, 30]. Le tableau revient à son état initial.
