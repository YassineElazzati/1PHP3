01
Le concept

Concrètement, PHP est un cuisinier invisible.

Pas un langage parmi d'autres. Pas un truc abstrait. C'est un moteur qui prépare une page web pour vous, sur un serveur, avant que votre navigateur n'en voie la première miette.

Le mot PHP veut dire « PHP : Hypertext Preprocessor ». Retenez juste preprocessor : ça pré-traite quelque chose. Ce qu'il pré-traite, c'est la page HTML qui finit affichée chez vous.

Le mieux, c'est de le voir. Cliquez sur « Exécuter » : à gauche, le fichier .php tel que vous l'écrivez. À droite, ce que votre navigateur reçoit vraiment.

⚙ Côté serveur · source
<?php
$nom = "Marie";
$heure = date("H");
?>

<h1>Bonjour <?= $nom ?> !</h1>
<p>Il est <?= $heure ?>h.</p>
◆ Côté navigateur · reçu
// En attente d'exécution...
// Cliquez sur « Exécuter » pour voir
// ce que reçoit réellement le client.
Prêt
◆
Ce qu'il faut absolument retenir

Le code PHP n'existe pas dans le navigateur. Faites un clic droit → « Afficher le code source » sur n'importe quel site PHP : vous ne verrez jamais une seule balise <?php. Tout a été remplacé par du HTML, sur le serveur, avant l'envoi.

▲
L'erreur de débutant la plus fréquente

Croire que PHP s'exécute « quelque part dans le navigateur ». Non. Si vous fermez le serveur (Apache), votre fichier .php ne fait rien, même si vous l'ouvrez à la main dans Chrome. Le navigateur ne sait pas lire le PHP — il a besoin que quelqu'un (Apache + PHP) lui ait préparé du HTML d'abord.

03
Le paysage

Quatre technos, quatre rôles. Ne les confondez plus.

HTML, CSS, JavaScript, PHP. On les croise tous les jours, et on les mélange souvent. Voici la frontière claire entre eux : où chacun s'exécute, et pourquoi ça change tout.

◆ Client
HTML
Structure le contenu. Pose les titres, paragraphes, liens.
<h1>Bonjour</h1>
◆ Client
CSS
Donne l'apparence. Couleurs, polices, espacement.
h1 { color: red }
◆ Client
JavaScript
Anime, valide, interagit. Sans recharger la page.
btn.onclick = ...
⚙ Serveur
PHP
Décide quoi envoyer. Logique métier, BDD, sécurité.
<?php echo $user ?>
◇
Le test mental qui tranche

Posez-vous la question : « Est-ce qu'un utilisateur malin pourrait voir ce code en faisant clic droit ? » Si oui → c'est du client (HTML/CSS/JS). Si non → c'est du serveur (PHP). C'est pour ça qu'un mot de passe ou une vérification d'identité doit absolument se faire en PHP, jamais en JavaScript.

04
L'installation

Pour faire tourner PHP, il faut un atelier complet.

Vous ne pouvez pas simplement « installer PHP » et attendre que la magie opère. PHP a besoin d'un écosystème : un serveur pour l'appeler, une base pour ranger les données, des outils pour visualiser tout ça. C'est ce qu'on appelle un stack.

Plutôt que d'installer chaque brique séparément (douloureux), on utilise un package tout-en-un. Le plus connu s'appelle XAMPP. Regardez ce qui se cache derrière ce nom :

Sur Mac : MAMP, même idée

Sur macOS, l'équivalent s'appelle MAMP : Mac, Apache, MySQL, PHP. Mêmes briques, packaging légèrement différent. Les ports par défaut changent (8888 au lieu de 80), c'est tout.

▲
Le port 80 refuse de démarrer ?

Sur Windows, le port 80 est souvent squatté par Skype, IIS ou un antivirus. Si Apache plante au démarrage, ouvrez httpd.conf et remplacez Listen 80 par Listen 8080. Vos sites seront alors sur http://localhost:8080.

◇
Et en entreprise, on utilise quoi ?

Pas XAMPP. Les boîtes sérieuses (BlaBlaCar, Leboncoin…) utilisent Docker, qui crée des conteneurs identiques entre la machine du dev et le serveur de production. Mais c'est trop lourd pour apprendre — XAMPP est parfait pour l'école, Docker viendra en B3.

Si quelque chose plante

Les erreurs les plus fréquentes au premier démarrage, et comment les régler.

▲ Apache refuse de démarrer (Windows)
Le port 80 est utilisé par un autre programme (Skype, IIS, antivirus, World Wide Web Publishing Service).
Solution : dans XAMPP Control Panel → Config → Apache (httpd.conf) → remplacer Listen 80 par Listen 8080. Accédez ensuite à http://localhost:8080.
▲ MySQL refuse de démarrer
Une autre instance de MySQL tourne déjà sur le port 3306, ou un précédent crash a laissé des fichiers de verrouillage.
Solution : arrêter le service Windows MySQL (services.msc) ou supprimer les fichiers ibdata1 et ib_logfile* dans xampp/mysql/data.
▲ Erreur 404 sur test.php
Le fichier n'est pas dans htdocs, ou Windows l'a renommé en test.php.txt sans que vous le voyiez.
Solution : dans l'Explorateur Windows → Onglet Affichage → cochez « Extensions de noms de fichiers ». Renommez le fichier en test.php.
▲ Gatekeeper bloque MAMP (Mac)
macOS empêche par défaut l'ouverture d'applications téléchargées hors App Store.
Solution : faire un clic droit sur MAMP dans Applications → Ouvrir → confirmer Ouvrir dans le popup.
▲ Le code PHP s'affiche au lieu de s'exécuter
Vous avez ouvert le fichier directement (double-clic), au lieu de passer par localhost.
Solution : dans la barre du navigateur, tapez http://localhost/test.php (pas file:///...).
▲ Page blanche, aucune erreur
Une erreur PHP a planté le script, mais l'affichage des erreurs est désactivé.
Solution : ajouter en haut de votre fichier error_reporting(E_ALL); ini_set('display_errors', 1); pour voir les erreurs.
06
Les composants

Maintenant, qui fait quoi exactement.

Vous avez les briques, mais chacune a un rôle bien précis dans la chaîne. Survolez chaque carte pour voir la métaphore qui marche.

🚪

Apache

PORT 80 / 8080
Le portier du serveur. Il écoute les requêtes HTTP qui arrivent du navigateur. Selon ce qui est demandé, il sert un fichier directement (image, CSS) ou passe la main à PHP.
Comme le réceptionniste d'un hôtel qui oriente les clients.
⚙️

PHP

L'INTERPRÉTEUR
Reçoit le fichier .php d'Apache, lit le code ligne par ligne, l'exécute, et renvoie en sortie du HTML pur. C'est lui qui fait tout le « travail intellectuel ».
Le cuisinier qui suit la recette pour préparer l'assiette.
🗄️

MariaDB / MySQL

PORT 3306
La base de données. Stocke tout ce qui doit survivre au-delà d'une requête : utilisateurs, articles, commandes, messages. PHP lui parle en langage SQL.
Le garde-manger : grand, organisé, persistant.
📊

phpMyAdmin

LOCALHOST/PHPMYADMIN
Une interface web qui permet de voir et gérer la base MySQL sans écrire de SQL à la main. On y crée les tables, on ajoute des lignes, on teste des requêtes.
Le tableau d'inventaire avec stylo et papier.
📁

htdocs

C:\XAMPP\HTDOCS
Le dossier sacré. C'est le seul endroit où Apache va chercher les fichiers à servir. Un fichier hors de htdocs n'existe pas, du point de vue du web.
La salle du restaurant — hors de la salle, pas de service.
🎛️

XAMPP Control Panel

INTERFACE PRINCIPALE
Le tableau de bord avec un bouton Start/Stop pour chaque service. C'est par là qu'on démarre Apache et MySQL avant de commencer à coder.
La salle de contrôle de l'atelier.
07
Le dossier sacré

Pourquoi htdocs est le dossier le plus important.

C'est le piège n°1 du débutant : créer un super fichier PHP, l'enregistrer sur le Bureau, et se demander pourquoi localhost ne le trouve pas. Voyons exactement ce qui se passe.

Cliquez sur l'un des deux fichiers pour voir s'il est accessible via le navigateur :

📁 Dossier htdocs
C:\xampp\htdocs\
📄 index.php
📁 Bureau (n'importe où ailleurs)
C:\Users\Vous\Desktop\
📄 monsite.php
🌐 http://localhost/... EN ATTENTE
◆
La règle absolue

Pour qu'un fichier soit accessible via http://localhost, il doit être quelque part dans htdocs (ou un sous-dossier de htdocs). C'est ce qu'on appelle la racine documentaire. Apache ne regarde nulle part ailleurs.

08
Premier test

Vérifions que tout marche en 3 lignes.

Maintenant que l'atelier est monté, il faut le tester. On va écrire le plus petit script PHP possible, le placer au bon endroit, et regarder ce qui s'affiche dans le navigateur.

C:\xampp\htdocs\test.php
1<?php
2// Affiche toutes les infos de configuration PHP
3phpinfo();
4?>

Trois choses à remarquer sur ce micro-script :

  • Les balises <?php et ?> encadrent tout code PHP. En dehors d'elles, on est en HTML.
  • Le point-virgule ; à la fin de la ligne 3. C'est obligatoire — oubliez-le et PHP refuse de tourner.
  • La fonction phpinfo() est intégrée à PHP : elle génère toute seule une page de diagnostic.

Ouvrez votre navigateur, allez à http://localhost/test.php (ou http://localhost:8888/test.php pour MAMP). Vous devez voir une grande page violette pleine d'informations : version de PHP, modules chargés, configuration du serveur. Si vous la voyez, votre atelier est opérationnel.

▲
Trois bugs classiques au premier essai

Vous voyez le code source brut au lieu de la page violette → Apache n'est pas démarré. Retournez dans XAMPP Control Panel et cliquez Start.

Erreur 404 → le fichier n'est pas dans htdocs, ou il s'appelle test.php.txt (Windows cache les extensions par défaut, activez l'affichage).

Une erreur de syntaxe → vous avez oublié le point-virgule, ou une des balises.

09
Mise à l'épreuve

À vous de jouer. Cinq questions, vraies conditions.

Cliquez sur la réponse que vous pensez correcte. Vous aurez immédiatement le verdict, et une explication pour ancrer le réflexe.

Quelle est la balise d'ouverture correcte pour commencer un bloc de code PHP dans un document HTML ?
Réponse C. <?php est la seule balise d'ouverture standard. Les autres formes n'existent pas en PHP — elles peuvent ressembler à d'autres langages (JSP, ASP), mais ici elles déclencheraient une erreur de syntaxe.
Que signifie l'acronyme XAMPP, et quels composants regroupe-t-il pour le développement PHP local ?
Réponse B. Le X signifie « Cross-platform » (multiplateforme). Les quatre autres lettres correspondent aux logiciels installés ensemble : Apache (serveur web), MariaDB (base de données), PHP (interpréteur) et Perl (autre langage, rarement utilisé aujourd'hui).
Dans quel répertoire les fichiers PHP doivent-ils être placés au sein d'une installation XAMPP pour être accessibles via le navigateur ?
Réponse D. htdocs est la racine documentaire du serveur Apache. Tout fichier placé dedans devient accessible via http://localhost/. Hors de ce dossier, le fichier existe sur votre disque mais Apache ne le verra jamais.
Comment PHP traite-t-il une requête sur le serveur avant d'envoyer la réponse au navigateur ?
Réponse A. PHP est un langage côté serveur. Tout le traitement se fait sur le serveur, et le navigateur ne reçoit que le HTML produit. Le code PHP n'est jamais visible côté client, c'est cette propriété qui le distingue de JavaScript.
Quels grands sites web utilisent (ou ont utilisé) PHP, démontrant sa capacité à tenir un trafic massif ?
Réponse C. Facebook a été initialement écrit en PHP (puis en Hack, un dialecte de PHP). WordPress propulse 43 % de tous les sites du web. Wikipedia tourne sur MediaWiki, écrit en PHP. Google, Amazon et Netflix, eux, n'utilisent pas PHP comme cœur de stack.
