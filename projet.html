Architecture · Arborescence

L'arborescence du projet.

Voici la structure de fichiers à créer dans le dossier salon-coiffure/. Chaque fichier a un propriétaire — c'est lui qui doit le créer.

salon-coiffure/
├── config.php                  ← commun (tous)
├── index.php                   ← Personne B (accueil)
├── contact.php                 ← Personne C
├── reservations.php            ← Personne D (squelette) + B (listing)
├── ajouter-reservation.php     ← Personne A (CREATE)
├── modifier-reservation.php    ← Personne C (UPDATE)
├── supprimer-reservation.php   ← Personne D (DELETE)
├── includes/
│   ├── header.php              ← Personne A
│   └── footer.php              ← Personne A
└── assets/
    └── style.css               ← CSS commun (tous y ajoutent leurs styles)

S.4🤝 ENSEMBLE
10 min

Créer config.php (la connexion)

Le fichier qui contient la connexion PDO à la base. Sera require'd par toutes les pages qui touchent à la BDD.

Ce que le fichier doit contenir
  1. Les 4 constantes : DB_HOST ('localhost'), DB_NAME ('salon-coiffure'), DB_USER ('root'), DB_PASS ('' sur Windows / 'root' sur Mac MAMP)
  2. Un try / catch qui instancie new PDO(...) avec ces 3 paramètres : DSN, user, password
  3. Le DSN format : "mysql:host=...;dbname=...;charset=utf8mb4"
  4. Les options : PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION et PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC
  5. Le catch (PDOException $e) qui affiche un die() en cas d'erreur
Comment l'utiliser dans les autres fichiers
  1. Tout en haut de chaque fichier qui touche à la BDD, ajouter : require "config.php";
  2. Cette ligne charge le fichier et exécute son contenu, ce qui crée la variable $pdo
  3. Ensuite, dans le fichier, on peut utiliser $pdo->query(...) ou $pdo->prepare(...) librement
  4. Si le fichier est dans un sous-dossier (cas de includes/header.php), il faut remonter d'un niveau : require "../config.php";
💡 Référence

Reprendre le modèle vu dans le cours CRUD/PDO, section 2.3.

S.6🤝 ENSEMBLE
15 min

Gérer les branches Git pour travailler à 4 sans s'écraser

Travailler à 4 sur le même dossier sans Git = chaos garanti. La méthode propre : chacun travaille sur sa branche, puis on fusionne tout à la fin.

1. Mise en place (1 personne le fait, les autres clonent)
  1. Une personne crée un repo sur GitHub (privé ou public, peu importe) et ajoute les 3 autres comme collaborateurs (Settings → Collaborators)
  2. Cette personne initialise localement dans salon-coiffure/ : 
    git init
git add .
git commit -m "Setup initial"
git remote add origin https://github.com/.../salon-coiffure.git
git branch -M main
git push -u origin main
  3. Les 3 autres récupèrent le projet : 
    git clone https://github.com/.../salon-coiffure.git
cd salon-coiffure
2. Chacun crée et utilise SA branche
  1. Créer sa branche depuis main (à faire une seule fois) : 
    git checkout main
git pull
git checkout -b branche-A   # personne A — adapter avec votre lettre
  2. Coder normalement sur ses fichiers
  3. Sauvegarder régulièrement (toutes les 30 min ou après chaque tâche) : 
    git add .
git commit -m "A.1 : header.php créé"
git push origin branche-A
  4. Pour basculer entre branches : git checkout nom-branche
3. À la fin — fusionner toutes les branches dans main
  1. Tous poussent leurs derniers commits sur leur branche
  2. Revenir sur main et fusionner les 4 branches : 
    git checkout main
git pull
git merge branche-A
git merge branche-B
git merge branche-C
git merge branche-D
git push origin main
  3. S'il y a des conflits (deux personnes ont modifié le même fichier) : ouvrir le fichier dans VS Code, les zones en conflit sont marquées par <<<<<<< et >>>>>>> — choisir quelle version garder, puis git add . + git commit
💡 Astuce pratique

Avant de commencer à coder, toujours faire git pull pour récupérer ce que les autres ont peut-être déjà poussé.

⚠️ Si Git est trop compliqué

Plan B : partager le dossier via Google Drive / Dropbox / OneDrive et travailler à tour de rôle sur les fichiers communs (header, footer, reservations.php). Chacun crée ses propres fichiers en parallèle. Moins propre mais ça marche pour un projet court.

S.7🤝 ENSEMBLE
10 min

Mettre en place le CSS commun

On utilise Bootstrap pour la base (composants, grille, couleurs) et un fichier style.css custom pour les ajouts spécifiques (couleurs du salon, polices, animations…).

Étapes
  1. Créer le sous-dossier assets/ dans salon-coiffure/
  2. Créer un fichier assets/style.css vide pour l'instant — chacun y ajoutera ses styles au fil du projet
  3. Dans includes/header.php (créé par A), inclure les deux feuilles de style dans le <head> :
    • D'abord Bootstrap (CDN)
    • Ensuite assets/style.css (pour qu'il puisse surcharger Bootstrap si besoin)
  4. Forme du lien custom : <link rel="stylesheet" href="assets/style.css">
Conventions pour éviter les conflits sur style.css
  1. Chacun préfixe ses classes par sa lettre : .a-hero (Personne A), .b-card-service (Personne B), etc.
  2. Chacun écrit ses styles dans une section commentée du fichier : 
    /* ===== Personne A (header/footer) ===== */
.a-navbar { ... }

/* ===== Personne B (accueil) ===== */
.b-hero { ... }
  3. Comme ça, pas de conflit Git sur le même fichier (chacun touche à sa section)
💡 Astuce — démarrer sans CSS custom

Si Bootstrap seul suffit pour rendre le site joli (ce qui est souvent le cas), on peut laisser style.css vide et y revenir uniquement si on veut un détail particulier (couleur du salon, police…).

Formulaire d'ajout — ajouter-reservation.php

salon-coiffure/ajouter-reservation.php
Étapes

A.4A
30 min

Validation + INSERT en BDD

Étapes
  1. Détecter la soumission avec $_SERVER['REQUEST_METHOD'] === 'POST'
  2. Récupérer chaque champ depuis $_POST, les nettoyer avec trim()
  3. Vérifier que les champs obligatoires sont remplis (empty())
  4. Vérifier que l'email est valide (filter_var($email, FILTER_VALIDATE_EMAIL))
  5. Vérifier que le téléphone fait bien 10 caractères, que la date n'est pas dans le passé
  6. S'il n'y a aucune erreur : requête préparée INSERT INTO reservations (...) avec prepare() + execute()
  7. Rediriger vers reservations.php avec header("Location: ...") puis exit
  8. S'il y a des erreurs : ré-afficher le formulaire avec les messages d'erreur en rouge sous chaque champ, et les valeurs déjà saisies conservées
⚠️ Sécurité

Pour conserver les valeurs en cas d'erreur : value="<?= htmlspecialchars($_POST['nom'] ?? '') ?>". Sans htmlspecialchars = faille XSS.

A.5A
10 min

Tester de bout en bout

Modifier une réservation — modifier-reservation.php

salon-coiffure/modifier-reservation.php
