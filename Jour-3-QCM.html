Q1 Facile Bases PHP
Quelle est la balise d'ouverture correcte pour commencer un bloc de code PHP dans un document HTML ?
  • A.<php>
  • B.<script php>
  • C.<?php
  • D.<%php%>
✓ Réponse C. <?php est la balise d'ouverture standard pour les blocs de code PHP. Elle se ferme avec ?> (facultatif en fin de fichier).
Q2 Facile Bases PHP
Quel caractère doit obligatoirement se trouver à la fin de chaque instruction PHP pour indiquer qu'elle est terminée ?
  • A.Le caractère deux-points :
  • B.Le caractère point .
  • C.Le point-virgule ;
  • D.Le caractère virgule ,
✓ Réponse C. Chaque instruction PHP doit se terminer par un point-virgule ;. L'oublier provoque une Parse error.
Q3 Facile Variables
En PHP, quel caractère spécial doit précéder chaque nom de variable lors de sa déclaration ou de son utilisation ?
  • A.Le signe dièse #
  • B.Le signe dollar $
  • C.Le signe arobase @
  • D.Le signe esperluette &
✓ Réponse B. Toutes les variables PHP doivent commencer par $. Ex : $nom, $age, $total.
Q4 Facile Variables
Quelle construction PHP est couramment utilisée pour afficher du texte directement dans le navigateur ou le terminal ?
  • A.print_r()
  • B.write()
  • C.display()
  • D.echo
✓ Réponse D. echo est la construction de langage la plus couramment utilisée. Note : print_r() existe mais sert à inspecter des structures (tableaux, objets), pas pour l'affichage standard.
Q5 Facile Types de données
Quel type de données PHP attribue-t-il à une variable lorsque vous y stockez la valeur 3.14 sans guillemets ?
  • A.string
  • B.float
  • C.integer
  • D.boolean
✓ Réponse B. PHP attribue le type float (ou double) aux nombres décimaux. Avec des guillemets "3.14", ce serait une chaîne (string).
Q6 Facile Types de données
Quel sera le résultat de ce code qui concatène une chaîne avec un entier ?
<?php
$age = 25;
echo 'Age: ' . $age;
?>
  • A.Une erreur de type
  • B.null
  • C.L'entier 25 uniquement
  • D.La chaîne Age: 25
✓ Réponse D. PHP convertit automatiquement l'entier en chaîne lors de la concaténation avec l'opérateur . (point). C'est la conversion de type automatique (type juggling).
Q7 Facile Opérateurs
Quelle est la différence entre == et === en PHP ?
<?php
var_dump(0 == 'hello');
var_dump(1 == '1');
var_dump(1 === '1');
?>
  • A.Aucune différence, ils comparent valeur et type identiquement
  • B.=== est plus rapide car il ignore la conversion
  • C.== pour les chaînes, === pour les entiers uniquement
  • D.== compare avec conversion de type, === compare valeur ET type
✓ Réponse D. == = comparaison souple avec conversion. === = comparaison stricte (valeur ET type identiques). Préférez === par défaut.
⚠️ Piège : 0 == 'hello' renvoyait true avant PHP 8 — en PHP 8+, c'est false. Mais 1 == '1' reste true alors que 1 === '1' est false.
Q8 Facile Opérateurs
Quelle valeur renvoie cette expression ?
<?php
$result = 17 % 5;
echo $result;
?>
  • A.3
  • B.5
  • C.3.4
  • D.2
✓ Réponse D. % est l'opérateur modulo qui renvoie le reste de la division entière. 17 = 5 × 3 + 2 → le reste est 2.
Q9 Facile Structures de contrôle
Quel mot-clé PHP permet de définir une condition alternative quand le if initial est faux ?
  • A.elif
  • B.otherwise
  • C.else if ou elseif
  • D.elsif
✓ Réponse C. PHP accepte les deux : else if (deux mots) et elseif (un seul mot).
⚠️ Piège : elif existe en Python, pas en PHP. Et elsif en Ruby/Perl — ne pas confondre.
Q10 Facile Structures de contrôle
Quel sera le résultat affiché ?
<?php
$sum = 0;
for ($i = 1; $i <= 5; $i++) {
    $sum += $i;
}
echo $sum;
?>
  • A.15
  • B.10
  • C.20
  • D.5
✓ Réponse A. La boucle itère $i de 1 à 5 (inclus). Somme = 1+2+3+4+5 = 15.
⚠️ Piège : $i <= 5 inclut 5. Avec $i < 5, on s'arrêterait à 4 et la somme vaudrait 10.
Q11 Facile Structures de contrôle
Quelle est la différence entre while et do-while en PHP ?
  • A.while vérifie la condition avant le bloc, do-while s'exécute toujours au moins une fois
  • B.while peut être infinie mais do-while a un max d'itérations
  • C.do-while est plus rapide car saute la vérification initiale
  • D.while supporte break/continue mais pas do-while
✓ Réponse A. do-while place la condition après le bloc, donc le bloc est garanti de s'exécuter au moins une fois, même si la condition est fausse dès le départ.
Q42 Moyen Structures de contrôle
Quel sera le résultat de ce for qui parcourt un tableau ?
<?php
$names = ['Alice', 'Bob', 'Charlie'];
for ($i = 0; $i < count($names); $i++) {
    echo $names[$i] . ' ';
}
?>
  • A.Alice uniquement
  • B.Une erreur fatale
  • C.Alice Bob Charlie
  • D.0 1 2
✓ Réponse C. La boucle utilise count() comme borne et accède aux éléments via $names[$i]. Indices 0, 1, 2 → valeurs Alice, Bob, Charlie.
💡 Plus idiomatique : utiliser foreach ($names as $name) qui évite la gestion manuelle de l'index.
Q35 Facile Structure de projet
Quel est le principal avantage de séparer le code HTML d'en-tête et pied de page dans des fichiers include ?
  • A.Évite la duplication et garantit une mise en page cohérente sur toutes les pages
  • B.Rend le site plus rapide car le navigateur met en cache les fichiers inclus
  • C.Exigé par PHP 8.0, sinon le site ne s'affiche pas
  • D.Empêche les attaques XSS automatiquement
✓ Réponse A. DRY : Don't Repeat Yourself. Modifier le header dans header.php met à jour toutes les pages qui l'incluent.
Q36 Moyen Structure de projet
Quelle est la différence entre require et include ?
  • A.require produit une erreur fatale si le fichier est manquant, include juste un avertissement
  • B.require charge à la compilation, include à l'exécution
  • C.include peut charger des URL distantes, pas require
  • D.require est pour les fichiers PHP, include pour HTML/CSS/JS
✓ Réponse A. require = E_COMPILE_ERROR et arrêt du script si fichier manquant. include = E_WARNING et le script continue. Pour un fichier vital (config, fonctions), préférer require.
Q37 Moyen Structure de projet
Quelle est la structure de répertoires recommandée pour un projet PHP ?
  • A.Tous les fichiers PHP dans un seul répertoire racine
  • B.Chaque page dans son propre sous-répertoire avec ses CSS/JS
  • C.Répertoires séparés pour les includes, ressources publiques, config, avec un index.php en point d'entrée
  • D.Fichiers PHP hors de la racine web, accessibles uniquement via API
✓ Réponse C. Structure type : includes/ (fonctions partagées), public/css/ + public/js/, config/, et index.php comme point d'entrée. Améliore la maintenabilité.
Q38 Facile Environnement XAMPP

  • A.eXtended Application Management and PHP Processing

Q39 Moyen Environnement XAMPP
Dans quel répertoire les fichiers PHP doivent-ils être placés sous XAMPP pour être accessibles via le navigateur ?
  • A.xampp/php/
  • B.N'importe quel répertoire — XAMPP scanne tout
  • C.xampp/apache/

Q40 Moyen Modèle serveur PHP
Comment PHP traite-t-il une requête avant d'envoyer la réponse au navigateur ?
  • A.Le serveur interprète le code PHP, génère le HTML, et envoie uniquement le HTML au navigateur
  • B.Le code PHP est téléchargé et exécuté localement par un moteur JS intégré
  • C.PHP se compile en binaire que le navigateur exécute
  • D.Le serveur envoie code PHP + HTML pour traitement en parallèle
✓ Réponse A. PHP est traité côté serveur. Le navigateur ne voit jamais le code PHP — c'est pour ça qu'il est sûr d'y mettre des mots de passe (contrairement à JavaScript). Faites Clic droit → Code source sur une page PHP : vous ne verrez que du HTML.

Q12 Facile Fonctions
Quel mot-clé PHP permet de définir une fonction ?
  • A.def
  • B.func
  • C.method
  • D.function
✓ Réponse D. Le mot-clé function sert à déclarer une fonction. def existe en Python, func en Go/Swift.
Q13 Facile Fonctions
Quelle sera la valeur de retour de maximum(10, 3) ?
<?php
function maximum($a, $b) {
    if ($a > $b) {
        return $a;
    }
    return $b;
}
echo maximum(10, 3);
?>
  • A.13 (addition)
  • B.3 (la plus petite)
  • C.10 (la plus grande)
  • D.Une erreur de type de retour
✓ Réponse C. Puisque 10 > 3 est vrai, la fonction entre dans le if et retourne $a = 10. Le return arrête immédiatement la fonction.
Q14 Facile Tableaux
Quelle fonction PHP renvoie le nombre d'éléments d'un tableau ?
  • A.count()
  • B.size()
  • C.length()
  • D.total()
✓ Réponse A. count() renvoie le nombre d'éléments. sizeof() existe aussi en PHP, mais c'est juste un alias de count(). length n'existe pas (c'est en JS).
Q15 Facile Tableaux
Différence entre tableau indexé et tableau associatif ?
  • A.Les indexés ne peuvent stocker que des chaînes
  • B.Les indexés utilisent des clés numériques, les associatifs des clés nommées
  • C.Les associatifs sont plus rapides grâce aux tables de hachage
  • D.Les indexés sont immuables après création
✓ Réponse B. Indexé : [10, 20, 30] → clés 0, 1, 2 auto. Associatif : ["nom" => "Alice", "age" => 30] → clés personnalisées.
Q16 Facile Tableaux
Que contient $fruits après ce code ?
<?php
$fruits = ['apple', 'banana', 'cherry'];
array_push($fruits, 'date');
print_r($fruits);
?>
  • A.[apple, banana, cherry, date] — ajouté à la fin
  • B.[banana, cherry, date] — remplace le premier
  • C.Erreur — syntaxe courte incompatible avec array_push
  • D.[date, apple, banana, cherry] — ajouté au début
✓ Réponse A. array_push() ajoute à la fin. Alternative équivalente : $fruits[] = 'date'; (plus idiomatique en PHP).
⚠️ À ne pas confondre : array_unshift() ajoute au début, array_pop() retire la fin, array_shift() retire le début.
Q17 Facile Chaînes de caractères
Quel opérateur PHP permet de concaténer deux chaînes ?
  • A.Le point .
  • B.Le plus + (comme en JavaScript)
  • C.L'esperluette &
  • D.La double barre ||
✓ Réponse A. En PHP, on concatène avec le point (.), pas avec +.
⚠️ Piège classique : Si on vient de JS, on écrit $nom + " " + $prenom qui en PHP fait une addition numérique → résultat = 0. La bonne syntaxe : $nom . " " . $prenom.
Q18 Facile Chaînes de caractères
Différence essentielle entre guillemets simples et doubles ?
  • A.Les simples sont plus rapides mais limités à 256 caractères
  • B.Les doubles interprètent les variables et séquences d'échappement, les simples traitent littéralement
  • C.Les simples sont immuables, les doubles modifiables
  • D.Les doubles sont pour multiligne, les simples pour une ligne
✓ Réponse B. "Bonjour $nom\n" → interprète $nom et \n (retour à la ligne). 'Bonjour $nom\n' → affiche littéralement Bonjour $nom\n.
💡 Astuce : les guillemets simples sont (très légèrement) plus rapides car pas de parsing. Mais en pratique, utilisez ce qui est le plus lisible.
Q19 Facile Chaînes de caractères
Qu'affichera ce code utilisant explode() ?
<?php
$str = 'PHP is great';
$words = explode(' ', $str);
print_r($words);
?>
  • A.La chaîne PHP-is-great
  • B.Un tableau de 3 éléments : ['PHP', 'is', 'great']
  • C.Le nombre 3
  • D.Erreur — explode nécessite une regex
✓ Réponse B. explode(séparateur, chaîne) découpe une chaîne en tableau. L'inverse est implode(séparateur, tableau) qui recolle en chaîne.
Q20 Facile Gestion de fichiers
Quelle fonction lit l'intégralité d'un fichier dans une variable chaîne ?
  • A.fread() — nécessite descripteur + longueur
  • B.readfile() — affiche directement, ne retourne pas
  • C.file_get_contents() — lit tout en une opération
  • D.fgets() — lit une ligne, à boucler
✓ Réponse C. file_get_contents() = le moyen le plus simple pour lire un fichier entier. Son symétrique pour écrire : file_put_contents().
💡 Bonus : file() renvoie un tableau où chaque case est une ligne du fichier (utile pour traiter ligne par ligne).
Q21 Facile Gestion de fichiers
Rôle du drapeau FILE_APPEND dans file_put_contents() ?
  • A.Ajoute à la fin du fichier sans écraser le contenu existant
  • B.Crée une copie de sauvegarde avant l'écriture
  • C.Verrouille le fichier en exclusif
  • D.Valide les permissions avant écriture
✓ Réponse A. Sans FILE_APPEND, file_put_contents() écrase tout. Avec, il ajoute à la suite — indispensable pour un fichier de logs.
⚠️ Cas typique : oublier FILE_APPEND dans une fonction logAction() → seul le dernier appel reste dans le fichier.

Q44 Moyen Fonctions
Que va afficher ce code utilisant un paramètre par défaut ?
<?php
function saluer($nom = 'Monde') {
    echo 'Bonjour ' . $nom . ' ';
}
saluer();
saluer('Alice');
?>
  • A.Bonjour Monde deux fois
  • B.Bonjour Monde Bonjour Alice
  • C.Bonjour Bonjour Alice

⚠️ Règle : les paramètres avec valeur par défaut doivent être après les paramètres obligatoires. function f($a, $b = 5) ✓ — function f($a = 5, $b) ✗
Q45 Moyen Tableaux

  • B.[10, 20]

✓ Réponse A. Étape 1 : array_push → [10, 20, 30, 40]. Étape 2 : array_pop retire le dernier élément → [10, 20, 30]. Retour à l'état initial.

  • B.Erreur — clés associatives doivent être entières

Q47 Moyen Chaînes de caractères
Que va afficher ce code ?
<?php
$mot = 'Bonjour';
$longueur = strlen($mot);
echo "Le mot '$mot' contient $longueur lettres";
?>
  • A.Le mot 'Bonjour' contient 5 lettres
  • B.Le mot 'Bonjour' contient 8 lettres
  • C.Le mot 'Bonjour' contient 7 lettres
  • D.Erreur — strlen non utilisable en concaténation
✓ Réponse C. strlen('Bonjour') compte 7 caractères (B-o-n-j-o-u-r). Les variables $mot et $longueur sont interprétées dans la chaîne entre guillemets doubles.
⚠️ Piège UTF-8 : strlen('café') renvoie 5 et non 4, car é prend 2 octets en UTF-8. Pour compter les caractères, utiliser mb_strlen().
Q48 Moyen Gestion de fichiers
Rôle du second paramètre de fopen() et signification du mode 'w' ?
<?php
$fichier = fopen('donnees.txt', 'w');
fwrite($fichier, 'Nouvelle ligne');
fclose($fichier);
?>
  • A.w = "watch", surveille les modifications en temps réel
  • B.w ouvre et ajoute à la fin sans effacer
  • C.w ouvre en lecture seule
  • D.w ouvre en écriture, EFFACE le contenu existant ou crée le fichier s'il n'existe pas
✓ Réponse D. Les modes de fopen() : r=lecture, w=écriture (écrase), a=ajout (append, ne touche pas l'existant), r+=lecture+écriture.
⚠️ Catastrophe classique : ouvrir un fichier important en mode 'w' par erreur → tout son contenu est perdu instantanément. Pour ajouter à un log, utiliser le mode 'a'.

Q22 Facile Formulaires GET POST
Quel tableau superglobal contient les données soumises via un formulaire HTML en méthode POST ?
  • A.$_GET — stocke toutes les données quelle que soit la méthode
  • B.$_POST — données envoyées via la méthode HTTP POST
  • C.$_REQUEST — seul moyen d'accéder aux soumissions
  • D.$_FORM — dédié aux entrées de formulaires
✓ Réponse B. $_POST reçoit les données envoyées en POST.
⚠️ Pièges : $_REQUEST existe et regroupe GET+POST+COOKIE, mais c'est mal d'y recourir (sécurité, ambiguïté). $_FORM n'existe pas du tout.
Q23 Facile Formulaires GET POST
Différences principales entre GET et POST ?
  • A.GET est chiffré, POST est en texte brut visible
  • B.GET ajoute les données à l'URL (~2048 caractères max), POST les envoie dans le corps de la requête
  • C.POST est plus rapide car il saute la résolution DNS
  • D.GET supporte les téléversements de fichiers volumineux, POST se limite au texte
✓ Réponse B. GET → données visibles dans l'URL (?nom=Alice&age=30), limité à ~2 ko, utile pour les recherches et liens partageables. POST → données dans le corps de la requête, invisibles, pour les actions sensibles (login, paiement).
⚠️ Aucune des deux n'est chiffrée en soi — c'est HTTPS qui chiffre le tout.
Q24 Facile Formulaires GET POST
Que se passera-t-il lors d'une soumission sans username rempli ?
<?php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    if (empty($_POST['username'])) {
        echo 'Username is required';
    } else {
        echo 'Hello, ' . $_POST['username'];
    }
}
?>
  • A.Affichage Hello, !
  • B.Erreur fatale (variable POST non définie)
  • C.Username is required — empty() détecte la valeur manquante
  • D.Rien — PHP ignore les soumissions vides
✓ Réponse C. empty() renvoie true pour : "", "0", 0, null, false, [], et les variables non définies (sans warning). C'est ce qui en fait la fonction idéale pour valider un champ obligatoire.
Q25 Facile Validation des données
Quelle fonction protège contre les attaques XSS en encodant les caractères spéciaux pour l'affichage HTML ?
  • A.strip_tags() — supprime les balises mais n'encode pas
  • B.urlencode() — pour les URL uniquement
  • C.addslashes() — échappe quotes/antislashes
  • D.htmlspecialchars() — convertit les caractères spéciaux en entités HTML
✓ Réponse D. htmlspecialchars() transforme < en &lt;, > en &gt;, &, " et ' en entités. Les balises sont rendues visibles comme texte mais pas exécutées.
⚠️ Confusion classique : htmlspecialchars() ENCODE / strip_tags() SUPPRIME. Pour de l'affichage, on encode (on garde le contenu visible).
Q26 Facile PDO MySQL
Que signifie l'acronyme PDO et quel est son objectif principal ?
  • A.PHP Data Output — formate les résultats pour l'affichage HTML
  • B.PHP Data Objects — interface cohérente pour plusieurs SGBD
  • C.PHP Database Optimizer — optimise automatiquement les requêtes
  • D.PHP Direct Operation — manipule directement les fichiers de BDD
✓ Réponse B. PDO = PHP Data Objects. Couche d'abstraction qui parle MySQL, PostgreSQL, SQLite, Oracle, etc. avec la même syntaxe.
💡 Avantage clé : changer de SGBD = changer le DSN, le reste du code reste identique.
Q27 Facile PDO MySQL
Que fait ce code de connexion PDO avec gestion d'erreurs ?
<?php
try {
    $pdo = new PDO('mysql:host=localhost;dbname=webblog', 'root', '');
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    echo 'Connected successfully';
} catch (PDOException $e) {
    echo 'Connection failed: ' . $e->getMessage();
}
?>
  • A.Si la connexion échoue, le bloc catch affiche le message d'erreur
  • B.La connexion réussit toujours — PDO crée la BDD si elle n'existe pas
  • C.setAttribute est facultatif sans effet
  • D.Le DSN doit inclure un port sinon la connexion expire
✓ Réponse A. Le try/catch capture la PDOException levée si la connexion échoue.
⚠️ Crucial : sans PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION, les erreurs sont muettes ou en warnings, et catch ne déclenche jamais.
Q28 Facile PDO MySQL
Pourquoi utiliser des requêtes préparées plutôt que d'insérer directement des variables dans le SQL ?
  • A.Elles s'exécutent plus vite (sautent l'analyse SQL)
  • B.Elles empêchent l'injection SQL en séparant la logique SQL des données utilisateur
  • C.Elles sont obligatoires en MySQL 8.0
  • D.Elles chiffrent automatiquement les données envoyées
✓ Réponse B. Avec prepare(), la structure SQL est envoyée séparément des valeurs. Un attaquant ne peut plus injecter du code malveillant comme ' OR 1=1 -- qui modifierait la logique de la requête.
Q29 Facile Opérations CRUD
Que signifie l'acronyme CRUD ?
  • A.Connect, Read, Update, Disconnect
  • B.Create, Read, Update, Delete
  • C.Cache, Retrieve, Upload, Download
  • D.Compile, Run, Undo, Debug
✓ Réponse B. Les 4 opérations fondamentales sur des données : Create (INSERT), Read (SELECT), Update (UPDATE), Delete (DELETE). Toute application web qui manipule des données passe par ces 4 verbes.
Q49 Moyen Validation des données
Comportement de ce code de validation combinant isset et empty ?
<?php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    if (!isset($_POST['email']) || empty($_POST['email'])) {
        echo 'Champ requis';
    } else {
        echo 'Champ valide';
    }
}
?>
  • A.Toujours Champ valide
  • B.Erreur — isset et empty incompatibles
  • C.Ne s'exécute jamais — REQUEST_METHOD bloque tout
  • D.Champ requis si l'email est absent OU vide
✓ Réponse D. Combo robuste : !isset détecte le champ manquant, empty détecte la chaîne vide. L'un ou l'autre = invalide.
💡 En pratique : empty() seul suffit en général car il gère déjà les variables non définies (sans warning).
Q50 Moyen Validation des données
Que produit ce code utilisant htmlspecialchars sur une entrée avec balises HTML ?
<?php
$input = '<script>alert("XSS")</script>';
echo htmlspecialchars($input, ENT_QUOTES, 'UTF-8');
?>
  • A.Le navigateur exécute le script — htmlspecialchars n'affecte pas les <script>
  • B.Le texte est affiché littéralement avec les balises visibles — caractères encodés
  • C.Les balises sont supprimées, seul "alerte" s'affiche
  • D.Erreur — htmlspecialchars ne gère pas les balises script
✓ Réponse B. < devient &lt; et > devient &gt;. Le navigateur affiche <script>alert("XSS")</script> comme du texte, pas comme du code à exécuter.
⚠️ ENT_QUOTES : ce flag fait que htmlspecialchars encode AUSSI les guillemets simples ' (par défaut, seuls les doubles le sont). À toujours mettre.
Q51 Moyen PDO MySQL
Quels sont les 3 paramètres obligatoires de new PDO() ?
<?php
$dsn = 'mysql:host=localhost;dbname=mabase;charset=utf8';
$user = 'root';
$password = '';
$pdo = new PDO($dsn, $user, $password);
?>
  • A.Nom de BDD, numéro de port, chemin du fichier de config
  • B.Type de BDD, chemin absolu vers MySQL, jeu de caractères
  • C.Chaîne DSN (hôte + nom de base), nom d'utilisateur, mot de passe
  • D.Adresse IP, nom de table, clé de chiffrement
✓ Réponse C. La signature : new PDO($dsn, $user, $password, $options). Le DSN décrit COMMENT se connecter (driver + host + dbname + charset), $user et $password = identifiants.
Q52 Moyen PDO MySQL
Que fait ce code SELECT avec fetchAll ?
<?php
$stmt = $pdo->query('SELECT nom FROM users');
$users = $stmt->fetchAll(PDO::FETCH_ASSOC);
foreach ($users as $user) {
    echo $user['nom'] . ' ';
}
?>
  • A.Récupère tous les enregistrements de la table users et affiche chaque nom
  • B.Affiche uniquement le premier — fetchAll est limité à un résultat
  • C.Modifie les enregistrements car query exécute toujours des écritures
  • D.Erreur — fetchAll nécessite le nom de la table en paramètre
✓ Réponse A. fetchAll(PDO::FETCH_ASSOC) renvoie un tableau de tableaux associatifs (clés = noms de colonnes).
⚠️ fetch() vs fetchAll() : fetch() = UN seul enregistrement (utile pour SELECT avec WHERE id), fetchAll() = TOUS (pour les listes).
Q53 Moyen Opérations CRUD
Rôle des marqueurs :nom et :email dans cette requête préparée INSERT ?
<?php
$sql = 'INSERT INTO users (nom, email) VALUES (:nom, :email)';
$stmt = $pdo->prepare($sql);
$stmt->execute([
    ':nom' => 'Alice',
    ':email' => 'alice@example.com'
]);
?>
  • A.Ce sont des commentaires SQL ignorés à l'exécution
  • B.Ils définissent les types de colonnes
  • C.Ils créent automatiquement les colonnes si manquantes
  • D.Ils remplacent les valeurs réelles via execute pour prévenir les injections SQL
✓ Réponse D. Les marqueurs nommés sont des placeholders. PDO les remplace par les valeurs du tableau passé à execute(), en s'assurant qu'aucun caractère ne puisse modifier la structure SQL.
💡 Alternative : les marqueurs positionnels ? fonctionnent aussi : execute(['Alice', 'alice@example.com']) (sans clés, ordre = celui des ?).
Q54 Moyen Opérations CRUD
Que fait cette requête UPDATE ?
<?php
$sql = 'UPDATE users SET nom = :nom WHERE id = :id';
$stmt = $pdo->prepare($sql);
$stmt->execute([':nom' => 'Bob', ':id' => 1]);
?>
  • A.Change le nom de l'utilisateur d'id = 1 en Bob
  • B.Crée un nouvel utilisateur — UPDATE insère si id absent
  • C.Supprime l'utilisateur d'id 1 — UPDATE = DELETE en SQL
  • D.Modifie tous les utilisateurs en les renommant Bob
✓ Réponse A. WHERE id = :id cible précisément la ligne d'id 1.
🚨 ATTENTION CATASTROPHE : sans WHERE, UPDATE users SET nom = 'Bob' modifierait TOUTES les lignes. Idem pour DELETE FROM users sans WHERE = toute la table effacée. Toujours vérifier la présence du WHERE !
Q58 Moyen Gestion des erreurs
Comportement de ce code avec try-catch ?
<?php
function diviser($a, $b) {
    if ($b === 0) {
        throw new Exception('Division par zéro impossible');
    }
    return $a / $b;
}
try {
    echo diviser(10, 0);
} catch (Exception $e) {
    echo $e->getMessage();
}
?>
  • A.Le script s'arrête avec une erreur fatale
  • B.Affiche 0 (PHP renvoie automatiquement 0)
  • C.Affiche Résultat affiché — finally s'exécute avant catch
  • D.Le catch intercepte l'exception et affiche le message sans arrêt
✓ Réponse D. throw lance une exception. Si elle est dans un try, le catch la capture et le script CONTINUE après le catch (au lieu de planter). $e->getMessage() récupère le texte de l'exception.
Q59 Moyen Bonnes pratiques de sécurité
Pourquoi ce code est-il vulnérable aux injections SQL ?
<?php
$email = $_POST['email'];
$sql = "SELECT * FROM users WHERE email = '$email'";
$result = $pdo->query($sql);
?>
  • A.Sécurisé — les guillemets simples empêchent toute injection
  • B.Sûr — PDO chiffre automatiquement les requêtes
  • C.Vulnérable seulement si ERRMODE_EXCEPTION n'est pas activé
  • D.L'insertion directe d'une variable dans le SQL permet à un attaquant de modifier la structure de la requête
✓ Réponse D. Si un attaquant tape ' OR 1=1 -- dans le champ email, la requête devient SELECT * FROM users WHERE email = '' OR 1=1 --' → la condition est toujours vraie, toutes les lignes sortent.
🛡️ Solution : requête préparée — $pdo->prepare("SELECT * FROM users WHERE email = ?") puis $stmt->execute([$email]). La valeur ne peut JAMAIS modifier la structure SQL.
